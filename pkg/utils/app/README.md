## App Configuration

This application uses a configuration management system to load and manage settings such as environment variables, database connections, Redis, S3, and APM configuration. The configuration is loaded from an `.env` file using the `viper` library.

### Configuration Struct: `AppConfig`

The `AppConfig` struct contains various settings required by the application, and its fields are mapped to the environment variables using `viper`. Below is an explanation of each section:

#### Environment Configuration:

* `ENVIRONMENT`: The environment the application is running in (e.g., "development", "production").
* `SERVER_ADDRESS`: The address where the server will listen for incoming connections.
* `CORS_ORIGIN`: Specifies the allowed origins for Cross-Origin Resource Sharing (CORS).
* `SERVICE_HOST_NAME`: The host name for the service.

#### Database Configuration:

* `DATABASE_DRIVER`: The database driver to use (e.g., "postgres", "mysql").
* `DATABASE_HOST`: The host of the database server.
* `DATABASE_PORT`: The port on which the database server is listening.
* `DATABASE_USERNAME`: The username used for database authentication.
* `DATABASE_PASSWORD`: The password used for database authentication.
* `DATABASE_NAME`: The name of the database to connect to.

#### Redis Configuration:

* `REDIS_HOST`: The host of the Redis server.
* `REDIS_PASSWORD`: The password for authenticating with the Redis server.
* `REDIS_MAX_IDLE`: The maximum number of idle connections to Redis.
* `REDIS_MAX_ACTIVE`: The maximum number of active connections to Redis.
* `REDIS_IDLE_TIMEOUT`: The timeout duration for idle Redis connections.

#### S3 Configuration:

* `BUCKET`: The name of the S3 bucket.
* `ACCESS_KEY_ID`: The access key ID for S3.
* `ACCESS_KEY_SECRET`: The access key secret for S3.
* `S3_REGION`: The region of the S3 service.
* `S3_HOST_NAME`: The host name of the S3 service.

#### APM (Application Performance Monitoring):

* `ELASTIC_APM_SERVER_URL`: The URL of the APM server (e.g., Elastic APM).
* `ELASTIC_APM_SERVICE_NAME`: The name of the service to report to the APM server.
* `ELASTIC_APM_ENVIRONMENT`: The environment of the service (e.g., "production").
* `ELASTIC_APM_SECRET_TOKEN`: The secret token for authenticating with the APM server.

### Function: `LoadAppConfig`

The `LoadAppConfig` function is responsible for loading the application configuration from an `.env` file using the `viper` library. It reads the configuration file, parses it, and then unmarshals it into the `AppConfig` struct.

#### Parameters:

* `path` (string): The path to the directory containing the configuration file (e.g., `"config"`).

#### Return:

* `config` (AppConfig): The application configuration containing all the necessary settings.
* `err` (error): Any error that occurred during the loading or unmarshaling process.

#### Example `.env` File:

```env
GO_ENV=production
SERVER_ADDRESS=0.0.0.0:8080
CORS_ORIGIN=https://example.com
SERVICE_HOST_NAME=my-service

DATABASE_DRIVER=postgres

DATABASE_PASSWORD=password
DATABASE_NAME=mydb

REDIS_HOST=localhost
REDIS_PASSWORD=redispassword
REDIS_MAX_IDLE=10
REDIS_MAX_ACTIVE=20
REDIS_IDLE_TIMEOUT=300s

BUCKET=my-bucket
ACCESS_KEY_ID=my-access-key-id
ACCESS_KEY_SECRET=my-access-key-secret
S3_REGION=us-east-1
S3_HOST_NAME=s3.amazonaws.com

ELASTIC_APM_SERVER_URL=http://localhost:8200
ELASTIC_APM_SERVICE_NAME=my-service
ELASTIC_APM_ENVIRONMENT=production
ELASTIC_APM_SECRET_TOKEN=my-secret-token
```

### Additional Notes:

* The `viper` library automatically loads environment variables and can also read configuration files like `.env` or `yaml`.
* The `.env` file should be placed in the root directory or the directory specified in the `configPath`.
* Ensure that all required environment variables are defined in the `.env` file or the system environment.

## Database Connection Setup

This application uses PostgreSQL as its database, and the `Connect` function is responsible for establishing a connection pool to PostgreSQL and providing access to the database queries.

### `Connect` Function

The `Connect` function establishes a connection to the PostgreSQL database, creates a connection pool, and returns the relevant database query interface and the connection pool object. It uses the `pgx` library for handling the PostgreSQL connection and `sqlc` for generating query interfaces.

#### Function Definition:

```go
func Connect(config AppConfig) (*dbCon.Queries, *dbCon.Store, *pgxpool.Pool)
```

#### Parameters:

* `config` (AppConfig): The application configuration containing the database credentials such as host, username, password, port, and database name.

#### Return:

* `*dbCon.Queries`: The `Queries` object generated by `sqlc` which provides methods to interact with the database.
* `*dbCon.Store`: A store object that can be used for executing queries and transactions.
* `*pgxpool.Pool`: The PostgreSQL connection pool that is used to manage database connections efficiently.

### How It Works:

1. **Database URL Construction:**

   * The database URL is dynamically constructed using the configuration values from `AppConfig`. It uses the `DATABASE_USERNAME`, `DATABASE_PASSWORD`, `DATABASE_HOST`, `DATABASE_PORT`, and `DATABASE_NAME` from the environment to form a connection string.

2. **Connection Pool:**

   * The function uses `pgxpool` to create a connection pool. It configures the pool with:

     * A maximum of 20 connections (`MaxConns`).
     * A minimum of 1 connection (`MinConns`).
     * A health check period of 5 seconds (`HealthCheckPeriod`).

3. **Store and Queries:**

   * Once the connection pool is successfully created, the function initializes `dbCon.NewStore` (which manages transactions and stores) and `dbCon.New` (which provides query methods). Both are returned for further use in the application.

4. **Error Handling:**

   * If any error occurs while parsing the connection string or creating the pool, the function logs the error and terminates the application using `log.Fatalf`.

### Configuration Example:

In the configuration (`AppConfig`), you will need to specify the following database details:

```go
type AppConfig struct {
	// Database
	DATABASE_USERNAME string
	DATABASE_PASSWORD string
	DATABASE_HOST     string
	DATABASE_PORT     string
	DATABASE_NAME     string
}
```

### Notes:

* Ensure that you have PostgreSQL running and accessible with the provided credentials.
* Adjust the maximum and minimum connection pool values based on your application’s expected load.
* The function automatically disables SSL (`sslmode=disable`) in the connection string, which can be modified if SSL is required for your environment.

## Redis Setup and Functions

This package provides functions to interact with a Redis server using a connection pool, enabling efficient management of Redis operations such as setting and getting keys, checking key existence, and batch deletion.

### Redis Connection Pool Setup

The Redis connection is managed using the `redigo` library, which provides a pool of Redis connections. The `RSetup` function is used to initialize the Redis connection pool based on the configuration provided.

#### `RSetup` Function

```go
func RSetup(config AppConfig) error
```

#### Parameters:

* `config` (AppConfig): The application configuration containing Redis connection parameters such as host, password, and connection pool limits.

#### How It Works:

* Initializes the Redis connection pool with the specified settings.

  * `MaxIdle`: Maximum number of idle connections in the pool.
  * `MaxActive`: Maximum number of active connections in the pool.
  * `IdleTimeout`: Duration for which idle connections are kept open.
* The connection is established using `redis.Dial`, and if a password is provided, it is authenticated using the `AUTH` command.
* The connection pool will be used by other Redis functions in the application.

### Redis Operations Functions

These functions interact with the Redis instance to perform common operations, such as setting, getting, checking existence, and deleting keys.

#### `RSet` - Set a Key/Value

```go
func RSet(key string, data interface{}, time int) error
```

* **Parameters**:

  * `key` (string): The Redis key to set.
  * `data` (interface{}): The data to be stored (it will be marshaled into JSON format).
  * `time` (int): The expiration time of the key in seconds.

* **How It Works**: This function serializes the data into JSON and sets it as the value for the specified Redis key. It also sets the expiration time for the key.

#### `RExists` - Check if a Key Exists

```go
func RExists(key string) (bool, error)
```

* **Parameters**:

  * `key` (string): The Redis key to check for existence.

* **Return**:

  * `true` if the key exists, `false` otherwise.
  * `err` for any errors encountered during the Redis operation.

#### `RGet` - Get a Key's Value

```go
func RGet(key string) ([]byte, error)
```

* **Parameters**:

  * `key` (string): The Redis key to retrieve.

* **Return**:

  * The value of the key in raw byte format, or an error if the key does not exist or the operation fails.

#### `RDelete` - Delete a Key

```go
func RDelete(key string) (bool, error)
```

* **Parameters**:

  * `key` (string): The Redis key to delete.

* **Return**:

  * `true` if the key was successfully deleted, `false` if not.
  * `err` for any errors encountered during the deletion process.

#### `RLikeDeletes` - Batch Delete Keys Using a Pattern

```go
func RLikeDeletes(key string) error
```

* **Parameters**:

  * `key` (string): The pattern to search for in Redis keys (supports wildcards like `*`).

* **How It Works**: This function finds all Redis keys that match the given pattern and deletes them.

#### `RGetValue` - Get a Key's Value and Deserialize It

```go
func RGetValue[T any](key string, defaultValue T) (T, error)
```

* **Parameters**:

  * `key` (string): The Redis key to retrieve.
  * `defaultValue` (T): The default value to return if the key doesn't exist or there is an error.

* **Return**:

  * The value of the key (deserialized into the provided type `T`), or the `defaultValue` in case of error.

* **How It Works**: This function retrieves the value of the key from Redis and deserializes it into the specified type (`T`). If an error occurs or the key doesn't exist, the default value is returned.

### Example `.env` Configuration

The Redis connection settings are pulled from the application's configuration (`AppConfig`), which can be defined in an `.env` file like this:

```env
REDIS_HOST=localhost:6379
REDIS_PASSWORD=your-redis-password
REDIS_MAX_IDLE=10
REDIS_MAX_ACTIVE=20
REDIS_IDLE_TIMEOUT=300s
```

### Additional Notes:

* **Connection Pooling**: The Redis connection pool is used to efficiently manage Redis connections, reducing overhead by reusing connections.
* **Error Handling**: Each function checks for errors (e.g., connection errors, key existence, etc.) and returns appropriate error messages.
* **JSON Serialization**: Data is serialized into JSON format before being stored in Redis and deserialized when retrieved.
